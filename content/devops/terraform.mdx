## Аналогия

<AnalogyBox>
Архитектор держит в руках план дома и сверяет его с тем, что стоит на участке. Чего нет по плану — построит, что стоит лишнее — снесёт, что сдвинулось — поправит. Тебе не нужно объяснять ему порядок работ: ты говоришь «хочу вот такой дом», а он сам разбирается, что и в каком порядке делать. Если двое рабочих попробуют перестроить один и тот же угол одновременно — план заблокируется, пока первый не закончит.
</AnalogyBox>

## Ключевые концепции

<ConceptGrid>
  <ConceptCard title="Provider" icon="Plug">
    Драйвер для конкретного облака или сервиса. `hashicorp/aws` знает как разговаривать с AWS API. Без provider-блока Terraform не знает, куда идти. Версию фиксируй — обновления ломают ресурсы.
  </ConceptCard>

  <ConceptCard title="Resource" icon="Server">
    Описание одного объекта инфраструктуры: S3-бакет, EC2-инстанс, DNS-запись. Terraform сравнивает описание с реальностью и доводит реальность до описания. Один ресурс — одна единица управления.
  </ConceptCard>

  <ConceptCard title="State" icon="Database">
    Файл `terraform.tfstate` — единственный источник правды о том, что Terraform уже создал. Local state опасен в команде: двое применят изменения поверх друг друга и получат конфликт. Храни state удалённо.
  </ConceptCard>

  <ConceptCard title="Plan / Apply" icon="GitBranch">
    `plan` показывает diff между state и конфигом — что создастся, изменится, удалится. `apply` исполняет этот diff. Никогда не делай `apply` без просмотра `plan` — удаление ресурса видно только там.
  </ConceptCard>

  <ConceptCard title="Module" icon="Layers">
    Переиспользуемый блок конфигурации — папка с `.tf`-файлами, которую вызываешь как функцию с параметрами. Вместо копирования одного и того же VPC в три окружения — один модуль с переменными `env`, `cidr`.
  </ConceptCard>

  <ConceptCard title="Backend + Lock" icon="Lock">
    Backend определяет, где хранится state. S3 + DynamoDB — стандарт для AWS: S3 хранит файл, DynamoDB блокирует его на время `apply`. Без блокировки параллельный `apply` от коллеги повредит state.
  </ConceptCard>
</ConceptGrid>

## Примеры

**main.tf — S3-бакет и DynamoDB для remote state**

```hcl
terraform {
  required_version = ">= 1.9"

  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
}

provider "aws" {
  region = "eu-central-1"
}

# Бакет для хранения tfstate других проектов
resource "aws_s3_bucket" "tfstate" {
  bucket = "my-company-tfstate"

  # Случайное удаление бакета уничтожит весь state — запрещаем
  lifecycle {
    prevent_destroy = true
  }
}

resource "aws_s3_bucket_versioning" "tfstate" {
  bucket = aws_s3_bucket.tfstate.id

  versioning_configuration {
    status = "Enabled"
  }
}

resource "aws_s3_bucket_server_side_encryption_configuration" "tfstate" {
  bucket = aws_s3_bucket.tfstate.id

  rule {
    apply_server_side_encryption_by_default {
      sse_algorithm = "AES256"
    }
  }
}

# Таблица для блокировки: без неё параллельный apply ломает state
resource "aws_dynamodb_table" "tfstate_lock" {
  name         = "my-company-tfstate-lock"
  billing_mode = "PAY_PER_REQUEST"
  hash_key     = "LockID"

  attribute {
    name = "LockID"
    type = "S"
  }
}
```

**backend.tf — подключение remote state**

```hcl
terraform {
  backend "s3" {
    bucket         = "my-company-tfstate"
    key            = "prod/network/terraform.tfstate"
    region         = "eu-central-1"
    dynamodb_table = "my-company-tfstate-lock"
    encrypt        = true
  }
}
```

**Основные команды**

```bash
# Инициализация: скачивает провайдеры, настраивает backend
terraform init

# Показывает diff — смотри внимательно перед apply
terraform plan

# Применяет изменения из последнего plan
# -auto-approve убирает интерактивный yes/no (только в CI)
terraform apply
terraform apply -auto-approve

# Удаляет всё что в state — осторожно на проде
terraform destroy

# Форматирование файлов по стандарту HashiCorp
terraform fmt -recursive

# Импорт существующего ресурса в state без пересоздания
terraform import aws_s3_bucket.legacy my-existing-bucket-name

# Посмотреть что сейчас в state
terraform state list
terraform state show aws_s3_bucket.tfstate
```
