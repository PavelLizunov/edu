> **Как использовать**: открой claude.ai (или Claude Design), вставь весь текст ниже в новый чат, попроси на выходе: палитру цветов в формате CSS-переменных, типографику, spacing-токены, и моки трёх ключевых экранов (главная, страница темы, мобильный вид). ## Проект

`edu.ninitux.com` — образовательная wiki-шпаргалка для инженеров уровня Middle+, готовящихся к повышению квалификации. Стартовый набор тем — DevOps (Ansible/Docker, Terraform, Kubernetes, сети, TLS, PostgreSQL, MongoDB, ELK). В дальнейшем расширяется на Backend, System Design, Algorithms.

Часть экосистемы `ninitux.com` — главный продукт владельца это **Virtual Penguin Network**, опенсорсный VPN-роутер (https://github.com/PavelLizunov/ninitux-landing). Эстетика бренда: **бруталистский минимализм, моноширинный код, тёмная тема по умолчанию, пингвин как мейнскот, технический документационный стиль без декоративности**.

## Целевая аудитория

Инженер 3–7 лет опыта, читает технические доки на работе, привык к Github / MDN / kubernetes.io. Ценит плотность информации, точность, скорость загрузки. Презрительно относится к "marketing-y" сайтам с большими CTA и stock-фото.

## Контент-юнит

Каждая тема — один MDX-файл со строгой структурой:

1. **Аналогия** — бытовое объяснение в 2–4 предложения. Выделенный блок (компонент `AnalogyBox`).
2. **4–6 концепций** в карточках с иконкой (компонент `ConceptCard`). Сетка.
3. **Примеры кода** — реальные команды/конфиги, с подсветкой и кнопкой "копировать" (компонент `CodeBlock`).
4. **Мини-квиз** — 1 вопрос, 3 варианта, объяснение после ответа (компонент `QuizCard`).

## Структура сайта

- **Главная (`/`)**: hero с подзаголовком → секция "Категории" (карточки) → список тем под раскрытой категорией → футер
- **Страница темы (`/[category]/[slug]`)**: хлебные крошки → заголовок темы → 4 секции (аналогия / концепции / код / квиз) → "следующая тема" внизу
- **Навигация на десктопе**: левый сайдбар, всегда виден, шириной ~280px, со списком категорий и тем
- **Навигация на мобиле**: нижняя панель с 3-4 иконками (Главная, Поиск (опц.), Темы, Тема day/night) + drawer-меню при тапе на "Темы"

## Что нужно от тебя

### 1. Цветовая палитра (формат — CSS variables для Tailwind v4)

Нужны токены для светлой и тёмной темы:
- `--background`, `--foreground`
- `--card`, `--card-foreground`
- `--popover`, `--popover-foreground`
- `--primary`, `--primary-foreground`
- `--secondary`, `--secondary-foreground`
- `--muted`, `--muted-foreground`
- `--accent`, `--accent-foreground` — для AnalogyBox (пингвин-блю или индиго)
- `--destructive`, `--destructive-foreground`
- `--border`, `--input`, `--ring`
- `--quiz-correct`, `--quiz-incorrect` — для feedback в квизе
- Один акцентный цвет, который **запоминается** и привязан к бренду (пингвин/Linux: рассмотри индиго #4F46E5–#6366F1, или penguin-orange #FF6B35, или electric blue #00B4FF)

**Тёмная тема — основной режим.** Светлая — переключаемая.

### 2. Типографика

- Заголовки и тело — system sans-serif стэк (без подгрузки), либо одна локальная переменная семейства (Inter / Geist / IBM Plex Sans — выбери)
- Код — моноширинный стэк (JetBrains Mono / Geist Mono / IBM Plex Mono)
- Размеры в Tailwind toolkit: text-xs (0.75rem) до text-5xl (3rem)
- Line-heights: для кода 1.5, для тела 1.65, для заголовков 1.15
- Letter-spacing: для заголовков -0.02em, для кода 0

### 3. Spacing и сетка

- Базовый spacing — Tailwind defaults (4px шаг)
- Контейнер: max-w-3xl (768px) для контентной части, max-w-7xl (1280px) для лейаута с сайдбаром
- Сайдбар: 280px фиксированной ширины
- Радиусы: `--radius: 0.5rem` (карточки, поля), `--radius-sm: 0.25rem` (бэйджи)

### 4. Компоненты — детали

#### `AnalogyBox`
Выделенный блок с аналогией. Должен **прочно бросаться в глаза, но не кричать**. Идеи:
- Фон `--accent` с opacity 0.1 + тонкая левая полоса `--accent` (4px)
- Или фон `--card` с иконкой лампочки слева (Lightbulb из lucide)
- Текст крупнее (text-lg), italic, но не слишком

#### `ConceptCard`
Карточка концепции, в сетке (на мобиле — 1 колонка, на md — 2 колонки, на lg — 3 колонки).
- Иконка (lucide) сверху, 32×32, цвет `--muted-foreground`
- Заголовок (text-base, font-semibold)
- Описание (text-sm, `--muted-foreground`)
- Hover — тонкое поднятие или подсветка border

#### `CodeBlock`
- Заголовок над блоком (если задан `label`)
- Подсветка через `shiki` (тема — `github-dark` для дарка, `github-light` для лайта; или одна универсальная)
- Кнопка "копировать" в правом верхнем углу, появляется на hover (на мобиле — всегда)
- Моноширинный шрифт, text-sm, padding 16px
- Горизонтальный скролл, не word-wrap
- Border + radius

#### `QuizCard`
- Вопрос крупно (text-lg, font-semibold)
- 3 кнопки-варианта вертикально (text-base, border, hover-эффект)
- После клика:
  - Правильный → border и фон `--quiz-correct` (мягкий зелёный)
  - Неправильный → border и фон `--quiz-incorrect` (мягкий красный)
  - Правильный (если выбран другой) → подсвечивается тоже
  - Под кнопками появляется блок объяснения с иконкой
- Анимация — лёгкая, не более 200ms
- После ответа нельзя перевыбрать (отключить кнопки)

### 7. Мобильный вид

- Сайдбар → drawer-меню (открывается с левого края или из нижней навигации)
- Хлебные крошки оставляем
- Сетка концепций — одна колонка
- Нижняя навигация: 3 иконки (Главная, Темы, Тема), всегда видна, sticky bottom

### 8. Что НЕ нужно

- ❌ Hero с большой картинкой
- ❌ "Trusted by" логотипы
- ❌ Анимированный фон, particles, gradient mesh
- ❌ Marketing-копирайт ("революция в обучении!")
- ❌ Stock-фото инженеров с ноутбуками
- ❌ Карусели / слайдеры

## Ограничения

- Tailwind CSS v4 (CSS-переменные нативно)
- shadcn/ui — берём что подходит (Button, Card, Badge, Sheet для drawer), стилизуем под токены
- Шрифты — локальные (`public/fonts/`), никаких Google Fonts CDN
- Поддержка `prefers-color-scheme` и `prefers-reduced-motion`
- Контраст WCAG AA минимум
